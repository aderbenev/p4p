Client Blocking API¶

This module provides Context for use in interactive and/or multi-threaded environment. Most methods will block the calling thread until a return is available, or an error occurs.

Usage¶

Start by creating a client Context.

>>> from p4p.client.thread import Context
>>> Context.providers()
['pva', ....]
>>> ctxt = Context('pva')

Get/Put¶

Get and Put operations can be performed on on single PVs or a list of PVs.

>>> V = ctxt.get('pv:name')
>>> A, B = ctxt.get(['pv:1', 'pv:2'])
>>> ctxt.put('pv:name', 5)
>>> ctxt.put('pv:name', {'value': 5}) # equivalent to previous

RPC¶

The RPC operation is similar to get+put except that the argument value must already be a Value.

from p4p import Type, Value
V = Value(Type([
    ('schema', 's'),
    ('path', 's'),
    ('query', ('s', None, [
        ('lhs', 'd'),
        ('rhs', 'd'),
    ])),
]), {
    'schema': 'pva',
    'path': 'pv:add',
    'query': {
        'lhs': 1,
        'rhs': 2,
    },
})
result = ctxt.rpc(V)

Monitor¶

Unlike get/put/rpc, the Context.monitor() method does not block. Instead it accepts a callback function which is called with each new Value, or Exception.

def cb(V):
       print 'New value', V
sub = ctxt.monitor('pv:name', cb)
time.sleep(10) # arbitrary wait
sub.close()

The monitor method returns a Subscription which has a close method to end the subscription.

API Reference¶

class p4p.client.thread.Context(*args, **kws)[source]¶

Context(provider, conf=None, useenv=True)

Parameters:
  • provider (str) – A Provider name. Try “pva” or run Context.providers() for a complete list.
  • dict (conf) – Configuration to pass to provider. Depends on provider selected.
  • bool (useenv) – Allow the provider to use configuration from the process environment.
  • int (maxsize) – Size of internal work queue used for monitor callbacks
  • unwrap – Controls Automatic Value unwrapping. Set False to disable

The methods of this Context will block the calling thread until completion or timeout

The meaning, and allowed keys, of the configuration dictionary depend on the provider.

The “pva” provider understands the following keys:

  • EPICS_PVA_ADDR_LIST
  • EPICS_PVA_AUTO_ADDR_LIST
  • EPICS_PVA_SERVER_PORT
  • EPICS_PVA_BROADCAST_PORT

>>> ctxt = Context('pva')
>>> ctxt.put('pv:name', 5.0)
>>> ctxt.put(['pv:1', 'pv:2'], [1.0, 2.0])
>>> ctxt.put('pv:name', {'value':5})
>>>

The provided value(s) will be automatically coearsed to the target type. If this is not possible then an Exception is raised/returned.

Unless the provided value is a dict, it is assumed to be a plan value and an attempt is made to store it in ‘.value’ field.
